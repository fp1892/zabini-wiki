Documentation Hub

This wiki serves as the central documentation space for the Zabini MVP system. It is intended to provide long-term, maintainable documentation for both technical maintainers and application users.

All documentation is protected by the same login mechanism as the MVP application itself. Access is intentionally restricted to ensure internal consistency and prevent misuse.

MVP Code Documentation

This section documents the complete technical implementation of the MVP system. It explains architectural decisions, data flow, authentication concepts, Firestore state management, admin functionality, undo mechanics and long-term maintenance considerations.

This documentation is written for developers, maintainers and technically inclined administrators. It is intentionally verbose and explanatory rather than tutorial-style.

MVP Application Documentation

This section describes the functional behavior of the MVP application itself. It explains all available actions, buttons, workflows and rules from a user and admin perspective.

The goal is to make the system transparent, predictable and fair for all participants. Every calculation and rule is documented explicitly.

Documentation Scope

This wiki intentionally avoids short summaries. Instead, it focuses on precise explanations, explicit assumptions and clear reasoning behind every system behavior.

  • No undocumented side effects
  • No hidden rules
  • No implicit assumptions
  • Long-term maintainability over brevity
