Purpose and Scope

The MVP application determines a fair and transparent MVP candidate based on historical performance and balancing rules.

1. Person Representation

Each participant is represented as a persistent entity. All historical data remains attached to the person until explicitly removed by an admin.

2. Event Recording

An event represents a single competitive session. For each event, up to ten placements can be recorded.

Participants without a placement automatically receive a default value to avoid bias.

3. MVP Cooldown Rule

The cooldown mechanism prevents the same person from becoming MVP repeatedly in short succession.

Cooldown is decremented only when MVP is awarded, not per event.

4. MVP Calculation

The MVP score is calculated using an intentionally simple formula. The goal is interpretability rather than mathematical optimization. 


averagePlacement = sum(placements) / numberOfPlacements

score =
  (20 - averagePlacement) * 3
  + (10 - mvpCount)
  - titlePenalty

5. Eligibility Rules

  • Blocked participants are excluded
  • Participants on cooldown are excluded
  • At least one recorded event is required

6. Administrative Actions

Administrative actions exist to recover from mistakes or migrate data. They are not intended for regular usage.

7. Expected Behavior

The system assumes good faith usage. All rules are deterministic and produce the same result for all users at all times.
