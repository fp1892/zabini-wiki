Purpose

The Hellcats MVP application determines the next MVP based on performance, fairness rules and historical results.

1. Person Entity

  • Name
  • Title flag (penalty applied)
  • Blocked flag (excluded from MVP)
  • MVP count
  • Cooldown counter
  • Event placements

2. Adding and Managing Persons

Add Person

Creates a new participant with default values.

Block / Unblock

Blocked persons are excluded from MVP calculations.

Title

A title applies a fixed score penalty to prevent repeat MVP dominance.

3. Events and Placements

Each event records placements from 1 to 10. All other participants automatically receive a default placement of 15.

4. MVP Cooldown

After receiving MVP, a person becomes temporarily ineligible. The cooldown decreases after each MVP assignment.

5. MVP Calculation


averagePlace = sum(placements) / count(placements)

score = (20 - averagePlace) * 3
        + (10 - mvpCount)

if title:
  score -= TITLE_PENALTY

6. Admin Features

Import JSON

Replaces the entire application state. Automatically creates an undo snapshot.

Reset

Clears all persons and events. Undo is available afterwards.

Undo

Restores the state before the last import or reset.

7. Expected Usage Rules

  • One event per session
  • Placements should be agreed upon before saving
  • Admin actions should be used carefully
